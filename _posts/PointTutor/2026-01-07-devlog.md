# TIL: Azure OpenAI 통합과 소소한 개선들 🤖✨
## 요약

오늘은 Azure OpenAI 서비스를 통합하고, 기존 코드베이스에 몇 가지 개선사항을 적용했습니다.  특히, 모델 추상화 레이어를 추가하여 향후 다른 LLM 서비스로 쉽게 전환할 수 있도록 설계했습니다.  오후에는 코드 전반적인 리팩토링과 함께 작은 버그들을 수정했습니다. 🐞

## 기술적 난관 🤯

Azure OpenAI 서비스를 처음 사용해보는 것이라서 인증, API 호출, 에러 핸들링 부분에서 어려움이 있었습니다. 특히 API 키 관리와 rate limiting을 고려한 retry 로직 구현에 시간을 많이 투자했습니다. 또한, 기존 코드베이스와 자연스럽게 통합하는 방식에 대한 고민도 많았습니다. 단순히 OpenAI API를 래핑하는 것 이상으로, 추상화 레이어를 둬서 향후 확장성을 고려해야 했기 때문입니다.

## 작업 상세 내용 🛠️

`406bf7d - feat: Various improvements and Azure OpenAI integration (toranan)` 커밋 하나에 담겨 있지만, 실제로는 여러 작업이 포함되어 있습니다.

*   **Azure OpenAI 통합**: Azure OpenAI 서비스 API를 호출하여 텍스트를 생성하는 기능을 추가했습니다. 인증, API 호출, 에러 핸들링을 포함한 전체적인 파이프라인을 구축했습니다.
*   **모델 추상화 레이어**: 향후 다른 LLM(Large Language Model) 서비스를 쉽게 통합할 수 있도록 모델 추상화 레이어를 추가했습니다. `OpenAIModel` 클래스를 만들고, 필요한 인터페이스를 정의했습니다. 이 덕분에 OpenAI뿐만 아니라 다른 LLM으로의 전환이 훨씬 용이해졌습니다.
*   **코드 정리 및 리팩토링**: 코드 전반적으로 불필요한 부분을 제거하고, 가독성을 높이기 위해 리팩토링을 진행했습니다.  타이포 수정 및 작은 버그들을 수정했습니다.

## 앞으로의 계획 🚀

*   **Azure OpenAI 성능 테스트 및 튜닝**: Azure OpenAI 서비스의 성능을 테스트하고, 최적의 파라미터를 찾아 튜닝할 예정입니다.
*   **더욱 풍부한 에러 핸들링**: 현재 에러 핸들링 로직을 보완하여 더욱 강력하고 사용자 친화적인 에러 메시지를 제공할 수 있도록 개선할 것입니다.
*   **테스트 코드 작성**: 통합된 기능에 대한 테스트 코드를 작성하여 안정성을 확보할 예정입니다.
*   **문서화 작업**: 변경된 부분에 대한 문서를 갱신하여 팀원들이 쉽게 이해하고 사용할 수 있도록 할 것입니다.

## 마무리 🙌

오늘은 Azure OpenAI 통합이라는 큰 숙제를 해결하면서도, 코드베이스 전체의 품질을 향상시키는 데 집중했습니다. 모델 추상화 레이어 덕분에 앞으로 LLM 관련 작업을 훨씬 유연하게 수행할 수 있을 것 같습니다.  다음 작업도 기대해주세요! 😊
