## 2025년 12월 첫째 주 개발 회고: AI 어시스턴트 모드 초기 구현과 인프라 안정화 삽질기 😅

이번 주는 신규 기능 구현과 인프라 안정화라는 두 마리 토끼를 잡으려다 햄스터 쳇바퀴 돌 듯 정신없이 지나갔습니다. 특히, AI 어시스턴트 모드 초기 구현에 많은 시간을 쏟았고, 예상치 못한 인프라 관련 문제들을 해결하느라 꽤나 애를 먹었네요. 그래도 한 주 동안 38개의 커밋을 쌓아 올리며 괄목할 만한 진전을 이뤄냈습니다. 자, 그럼 이번 주 개발 로그를 자세히 살펴볼까요?

**요약:**

*   AI 어시스턴트 모드의 핵심 기능 (힌트 제공) 구현 완료 및 백엔드 연동
*   학습 효율 증대를 위한 반복 학습 추천 알고리즘 초안 구현
*   Next.js와 관련 라이브러리 최신 버전 업데이트
*   Vercel 빌드 및 Supabase Storage 호환성 문제 해결
*   CloudAMQP RabbitMQ 연동 및 JVM 메모리 설정 최적화

**주요 작업:**

*   **✨ AI 어시스턴트 모드 초기 구현:**
    *   이번 주 가장 큰 작업은 AI 어시스턴트 모드의 초기 구현이었습니다. 문제 해결에 어려움을 겪는 사용자를 위해 힌트를 제공하는 기능을 개발했습니다.
    *   `/problems` 페이지에서 AI 어시스턴트 모드 토글을 추가하고, 오답 제출 시 힌트 API를 호출하여 힌트를 받을 수 있도록 구현했습니다.
    *   (커밋: `e95448f`, `05a7e5a`, `81ab117`)
*   **✨ 반복 학습 추천 알고리즘 (초안):**
    *   spaced repetition learning (간격 반복 학습) 알고리즘을 적용하여, 사용자의 학습 데이터를 기반으로 복습해야 할 문제들을 추천하는 기능의 초안을 구현했습니다. 이 기능을 통해 사용자들이 잊기 전에 중요한 개념들을 다시 복습하여 학습 효과를 극대화할 수 있도록 돕고자 합니다.
    *   (커밋: `04e7d1c`)
*   **✨ Python 크롤링 스크립트 통합:**
    *   dev2 브랜치에서 개발된 Python 크롤링 스크립트를 통합하여, 문제 데이터 수집 및 업데이트 프로세스를 자동화했습니다. (커밋: `1d2b321`)
*   **🐛 인프라 및 환경 설정 관련 버그 수정:**
    *   Next.js 업데이트 후 발생한 빌드 에러 및 Vercel 배포 문제를 해결했습니다. (`348a774`, `b4180bb`)
    *   Supabase Storage 연동 과정에서 발생한 경로 문제 및 권한 문제를 해결했습니다. (`ad8cf34`, `8bcd2c6`, `a0a788d`)
    *   CloudAMQP RabbitMQ 연동을 위한 SSL 설정 및 환경 변수 설정을 완료했습니다. (`6706822`, `58cff86`)
    *   Railway 배포 환경에서 JVM 메모리 부족으로 인해 발생하는 OOM (Out of Memory) 에러를 방지하기 위해 JVM 메모리 설정을 최적화했습니다. (`ac3a4dd`, `417b8a3`)
    *   CORS 에러를 해결하기 위해 허용된 Origin을 환경 변수를 통해 관리하도록 변경했습니다. (`38f0a29`, `729b438`, `2914c36`, `8c046a3`, `c580b8c`)
*   **♻️ 라이브러리 업데이트 및 의존성 관리:**
    *   Next.js와 React를 최신 버전으로 업데이트하여 성능 개선 및 최신 기능 활용 기반을 마련했습니다. (`4bd7dcf`)
    *   Next.js 19에서 더 이상 지원하지 않는 ESLint 설정을 제거했습니다. (`3c898ca`)
*   **🔒 보안 취약점 패치:**
    *   CVE-2025-66478 보안 취약점을 해결하기 위해 Next.js 버전을 15.4.8로 업데이트했습니다. 하지만, 다른 문제 발생으로 인해 다시 롤백했습니다. (`5a38c36`, `aafaf26`)

**기술적 도전:**

*   **Supabase Storage 연동:** 기존 파일 시스템 기반의 스토리지에서 Supabase Storage로 마이그레이션하면서 경로 처리 및 권한 설정에 많은 어려움이 있었습니다. 특히, production 환경에서 호환성 문제를 해결하는 데 시간이 오래 걸렸습니다.
*   **JVM 메모리 최적화:** Railway와 같은 제한된 환경에서 JVM 메모리를 효율적으로 관리하는 것은 까다로운 작업입니다. 특히, orchestrator 서비스의 메모리 사용량을 분석하고 적절한 제한을 설정하는 데 어려움을 겪었습니다.
*   **CORS 설정:** 다양한 환경 (로컬, 개발 서버, 프로덕션 서버)에서 CORS 문제를 해결하기 위해 환경 변수를 활용하여 유연하게 CORS 설정을 관리하도록 개선했습니다.

**다음 계획:**

*   AI 어시스턴트 모드 고도화: Gemini API 연동하여 더 정확하고 다양한 힌트 제공
*   반복 학습 추천 알고리즘 개선: 사용자 피드백 기반으로 알고리즘 개선 및 UI 개선
*   보안 취약점 해결: Next.js 업데이트 롤백으로 인해 발생한 보안 취약점 재검토 및 해결
*   테스트 자동화: E2E 테스트 및 Unit 테스트 추가하여 코드 안정성 확보
