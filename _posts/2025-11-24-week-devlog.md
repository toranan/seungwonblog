# AI 힌트 기능 강화 및 Gemini API 연동 개발 로그 (2025년 11월 24일 주간)
이번 주에는 MyPage 기능 강화와 AI 힌트 기능의 안정화, 그리고 Gemini API 연동을 위한 기반 작업에 집중했습니다. 다양한 기능 추가와 함께 보안 강화에도 신경 쓴 한 주였습니다. 꽤나 많은 커밋이 있었네요!

## 요약

* MyPage 사용자 경험 개선 (힌트 사용량 추이 및 성장 차트 추가)
* Gemini API 키를 환경 변수로 이동하여 보안 강화
* Supabase Storage 연동 및 Judge0 서버 업데이트
* UI 스타일 개선 및 버그 수정

## 주요 작업

### ✨ MyPage 기능 개선 및 데이터 연동 강화

*   **필터링된 힌트 사용 추이 및 성장 차트 추가:** MyPage에 사용자가 필터링한 힌트 사용량 추이를 보여주는 차트를 추가하여 사용 패턴을 더 쉽게 파악할 수 있도록 했습니다. (commit: `bd6a53f`)
*   **MyPage 데이터 및 CORS 문제 해결:** MyPage 데이터가 제대로 작동하도록 보장하고, CORS 관련 문제도 해결했습니다. (commit: `17ccf34`)
*   **AI 추천 새로고침 버튼 스타일 정렬:** MyPage 내 AI 추천 새로고침 버튼의 스타일을 정렬하여 사용자 인터페이스를 개선했습니다. (commit: `6cefcb9`)

### 🔒 Gemini API 연동 및 보안 강화

*   **Gemini API 키를 환경 변수로 이동:** 보안을 강화하기 위해 Gemini API 키를 코드에서 제거하고 환경 변수로 이동했습니다. (commit: `e18649f`)

### 📦 인프라 업데이트 및 기타 기능

*   **Supabase Storage 연동 및 Judge0 서버 업데이트:** 파일 저장 및 코드 실행 환경을 개선하기 위해 Supabase Storage를 연동하고 Judge0 서버를 업데이트했습니다. (commit: `67eaabf`)
*   **`is_visible` 컬럼 추가 및 포트 설정 수정:** 데이터베이스에 `is_visible` 컬럼을 추가하고, 포트 설정을 수정했습니다. (commit: `9ae4980`)

### 🐛 버그 수정 및 설정 오류 수정

*   **잘못된 문제 해결 작업 버튼 수정:** MyPage에서 잘못된 문제를 닫는 작업 버튼을 수정했습니다. (commit: `f3e0329`)
*   **docker-compose 환경 변수 경로 수정 및 MyPage 파일 복구:** docker-compose 환경 변수 경로를 수정하고, MyPage 파일을 초기 상태로 복구했습니다. (commit: `df0030d`)

*   **Supabase Storage 연동:** 기존 파일 저장 방식에서 Supabase Storage로 전환하면서 API 연동 및 파일 관리 전략을 변경해야 했습니다.
*   **Gemini API 키 보안 관리:** API 키를 안전하게 관리하기 위해 환경 변수를 사용하고, 보안 best practice를 준수해야 했습니다.
