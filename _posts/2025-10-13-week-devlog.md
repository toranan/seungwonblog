# 코드 실행 엔진 통합과 오케스트레이션 서비스 구축 - 2025년 10월 13일 개발 일지
이번 주에는 백엔드 성능 향상과 확장성을 위한 중요한 두 가지 작업을 완료했습니다. Judge0 코드 실행 엔진을 통합하여 다양한 언어를 지원하고, 백엔드 오케스트레이션 서비스인 "toranan"을 구축했습니다. 이 두 가지 작업은 앞으로 우리 서비스의 핵심 기능을 더욱 강력하게 만들어줄 것입니다.

## 요약

*   Judge0 코드 실행 엔진 통합으로 다국어 코드 실행 환경 구축
*   백엔드 오케스트레이션 서비스 "toranan" 구축

## 주요 작업

### ✨ 다국어 코드 실행 환경 구축 (feat: integrate Judge0 multi-language code execution system)

Judge0는 다양한 프로그래밍 언어를 지원하는 강력한 코드 실행 시스템입니다. 이 시스템을 통합함으로써 우리 서비스는 이제 Python, JavaScript, C++, Java 등 다양한 언어로 작성된 코드를 실행하고 결과를 얻을 수 있게 되었습니다.  단순히 코드 실행 뿐만 아니라, 사용자 코드를 안전하게 격리된 환경에서 실행하고, 실행 시간과 메모리 사용량을 제한하는 기능도 제공합니다. 이를 통해 더욱 안정적이고 효율적인 서비스를 제공할 수 있게 되었습니다.

### ✨ 백엔드 오케스트레이션 서비스 구축 (feat: add backend orchestrator service (toranan))

"toranan"은 마블 시네마틱 유니버스의 토르의 망치 "묠니르(Mjölnir)"에서 영감을 받아 만든 백엔드 오케스트레이션 서비스입니다. 마치 묠니르가 토르의 명령에 따라 번개를 소환하듯, "toranan"은 백엔드 서비스들의 실행, 관리, 모니터링을 중앙 집중적으로 처리합니다.  이를 통해 서비스 간의 의존성을 관리하고, 오류 발생 시 빠르게 대응할 수 있도록 돕습니다.  현재는 기본적인 기능만 구현되어 있지만, 앞으로 더 많은 기능을 추가하여 백엔드 시스템을 더욱 효율적으로 관리할 수 있도록 발전시켜 나갈 계획입니다.

## 기술적 도전

*   **Judge0 API 연동**: Judge0 API의 사용법을 익히고, 우리 서비스에 맞는 방식으로 통합하는 과정에서 API의 동작 방식을 깊이 이해해야 했습니다.  특히, 다양한 언어별로 코드 실행 환경을 설정하고, 에러 핸들링을 구현하는 부분이 까다로웠습니다.
*   **"toranan" 서비스 설계**: 오케스트레이션 서비스의 핵심 기능과 아키텍처를 설계하는 과정에서 확장성, 안정성, 유지보수성을 고려해야 했습니다.  특히, 서비스 간의 통신 방식과 데이터 일관성을 유지하는 방법을 고민하는 데 많은 시간을 할애했습니다.

## 다음 계획

*   **Judge0 security 강화**: Judge0의 security 설정을 강화하고, 사용자 코드 실행 환경을 더욱 안전하게 격리하는 방법을 연구할 예정입니다.
*   **"toranan" 대시보드 구현**: "toranan" 서비스의 상태를 시각적으로 보여주는 대시보드를 구현하여 운영 효율성을 높일 계획입니다.
*   **로그 시스템 연동**: Judge0와 "toranan"에서 발생하는 로그를 중앙 집중적으로 관리하고 분석할 수 있도록 로그 시스템을 연동할 예정입니다.

이번 주 작업은 서비스의 기반을 다지는 중요한 과정이었습니다. 앞으로도 꾸준히 노력하여 더욱 강력하고 안정적인 서비스를 만들어 나가겠습니다.
